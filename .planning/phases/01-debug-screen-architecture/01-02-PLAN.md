  <action>
Refactor the DebugScreen class to use a nested ScreenManager for sub-screen navigation.

**Keep unchanged:**
- PINKeypad, PINDialogContent, PINDialog classes (PIN entry)
- LargeSlider, SectionCard classes (reusable UI components)
- run_async helper function
- on_enter, _show_pin_dialog, _on_authenticated, _on_pin_cancelled methods

**Modify DebugScreen class:**

1. Add new imports at top:
   ```python
   from kivy.uix.screenmanager import ScreenManager, SlideTransition
   from monitoni.ui.debug_screens import BaseDebugSubScreen, DebugMenuScreen
   ```

2. Refactor `_build_ui` method:
   - Remove the old scrollable content building
   - Create `self.sub_screen_manager = ScreenManager(transition=SlideTransition(direction='left'))`
   - Create DebugMenuScreen with name='menu', passing navigate_callback=self.navigate_to, back_to_customer_callback=self.app.switch_to_customer
   - Add menu screen to sub_screen_manager
   - Create 7 placeholder sub-screens (one per category) using BaseDebugSubScreen:
     - Each with name matching menu item screen_name ('led', 'relay', 'sensor', 'audio', 'motor', 'network', 'stats')
     - Each with appropriate title ('LED Control', 'Relay Control', etc.)
     - Pass navigate_back=self.navigate_back to each
   - Add all sub-screens to sub_screen_manager
   - Set sub_screen_manager.current = 'menu'
   - Add sub_screen_manager to self (the Screen)

3. Add navigation methods:
   ```python
   def navigate_to(self, screen_name: str):
       """Navigate to a sub-screen."""
       self.sub_screen_manager.transition.direction = 'left'
       self.sub_screen_manager.current = screen_name

   def navigate_back(self):
       """Return to menu."""
       self.sub_screen_manager.transition.direction = 'right'
       self.sub_screen_manager.current = 'menu'
   ```

4. Keep all the section creation methods (_create_audio_section, _create_led_section, etc.) but comment them out or move to bottom - they will be migrated to sub-screens in Phase 2.

5. Keep all callback methods (_on_volume_change, _play_sound, etc.) - they will be used by sub-screens later.

**Important:** Use SlideTransition (not FadeTransition) for better performance on touchscreen hardware per RESEARCH.md recommendations.
