Modbus TCP transport layer, digital input controller, shared frame utilities, and dual-module config structure — foundation for switching from single RS485 relay to two Waveshare Ethernet relay modules.

## What Was Built

### modbus_utils.py

Shared Modbus RTU utilities used by all relay/DI controllers:
- `modbus_crc()` — extracted from modbus_relay.py (same algorithm)
- `build_write_coil_frame()` — FC05, 1-indexed channel to 0-indexed coil address
- `build_write_all_coils_frame()` — FC05 targeting Waveshare special address 0x00FF
- `build_read_coils_frame()` — FC01
- `build_read_discrete_inputs_frame()` — FC02

### modbus_tcp_relay.py — EthernetRelayController

Persistent TCP connection to Waveshare Ethernet relay modules (transparent mode):
- Raw Modbus RTU frames with CRC over TCP (no MBAP headers, no pymodbus)
- `asyncio.open_connection` + `readexactly(8)` for FC05 echo response
- Per-controller `asyncio.Lock`
- Background reconnect task with backoff [2, 5, 10, 30]s
- No command retry on failure (safety: avoids double-firing coils)
- In-memory `_relay_states` cache
- `connection_info` property for UI display

### modbus_digital_input.py — ModbusDigitalInputController + MockDigitalInputController

FC02 digital input reader over same TCP transparent-mode pattern:
- `read_digital_input(di_index)` — FC02, `readexactly(6)` for single-byte response
- Background poll loop at configurable interval (default 150ms)
- Door state callback on DI change
- Same reconnect pattern as EthernetRelayController
- `MockDigitalInputController` with 8-channel in-memory state, `simulate_door_open/close()`

### base.py — DigitalInputController ABC

New abstract class extending `SensorController`:
- Abstract `read_digital_input(di_index)` method
- Concrete `get_door_state()` → calls `read_digital_input(self._door_di_index)`
- Concrete `set_door_callback()` stores callback reference
- `_door_di_index` and `_door_callback` attributes

### config.py — EthernetRelayConfig, DoorSensorConfig

New pydantic models with full defaults for backward compat:
- `EthernetRelayConfig`: transport (tcp/serial), host, port, slave_address, timeout, serial_port, baudrate, max_channels
- `DoorSensorConfig`: method (gpio/modbus_di), gpio_pin, gpio_pull, gpio_active, di_index, poll_interval_ms
- `HardwareConfig` extended with `Optional` fields: relay_core, relay_levels, door_sensor

### config/default.yaml

New sections added under `hardware:`:
- `relay_core`: 8-CH Module C at 192.168.1.100, max_channels: 8
- `relay_levels`: 30-CH module at 192.168.1.101, max_channels: 30
- `door_sensor`: method: gpio (default), with modbus_di config ready for local.yaml override

Legacy `modbus:` and `gpio:` sections kept for backward compatibility.
