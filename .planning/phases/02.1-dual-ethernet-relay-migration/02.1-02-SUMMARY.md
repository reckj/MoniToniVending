# Phase 02.1 Plan 02: HardwareManager Dual Relay Routing Summary

**One-liner:** Dual relay routing in HardwareManager — relay_core for motor/spindle, relay_levels for door locks, with transport-aware init and config-driven sensor dispatch.

## What Was Built

Refactored `monitoni/hardware/manager.py` to support two independent relay module instances:

- **`relay_core`** — 8-CH Waveshare Module C (motor, spindle, digital inputs)
- **`relay_levels`** — 30-CH Waveshare module (door locks per vending level)
- **`relay` property** — backward-compat alias returning `relay_core`

### Initialization changes

| Method | What it does |
|--------|-------------|
| `_init_relay_core()` | TCP, serial, or legacy modbus fallback; mock on failure |
| `_init_relay_levels()` | TCP or serial; absent config -> mock (not an error) |
| `_init_sensor()` | Dispatches on `door_sensor.method`: `modbus_di` or `gpio` |

### Routing changes

| Convenience method | Old target | New target |
|-------------------|------------|-----------|
| `spin_motor()` | `self.relay` | `self.relay_core` |
| `unlock_door()` | `self.relay` | `self.relay_levels` |
| `lock_door()` | `self.relay` | `self.relay_levels` |

### Partial operation

Each module init is independently wrapped in try/except. If relay_core fails to connect, it falls back to MockRelayController and continues — relay_levels and sensor still initialize. The app can start with one real module down.

### Sensor dispatch

When `door_sensor.method == "modbus_di"`:
- Host/port taken from `config.hardware.relay_core` (DI is on the same 8-CH module)
- Creates `ModbusDigitalInputController` with configured `di_index` and `poll_interval_ms`
- Falls back to `MockDigitalInputController` on connection failure

When `door_sensor.method == "gpio"` (default/legacy):
- Uses existing `GPIOSensorController` path unchanged

### Shutdown

`shutdown()` now calls `stop_reconnect_loop()` on any `EthernetRelayController` instances before disconnect, preventing orphaned background tasks.

### Status reporting

`get_status()` reports `relay_core` and `relay_levels` separately. TCP controllers include their `connection_info` dict (host, port, status, last_error).
