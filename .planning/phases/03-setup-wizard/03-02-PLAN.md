**hardware_step.py** - HardwareStepScreen (Step 1, step_index=0):
- name='wizard_hardware'
- step_title="Hardware"
- step_description="Motor- und Relais-Grundeinstellungen"
- Essential settings (in a SettingsCard titled "Grundeinstellungen"):
  - Motor Relay Channel (NumpadField-style): config path "vending.motor.relay_channel", int, min=1, max=16
  - Spindle Lock Relay (NumpadField-style): config path "vending.motor.spindle_lock_relay", int, min=1, max=16
- Advanced settings (ExpandableSection titled "Erweiterte Einstellungen"):
  - Spin Delay ms: "vending.motor.spin_delay_ms", int, min=0, max=5000
  - Spindle Pre-Delay ms: "vending.motor.spindle_pre_delay_ms", int, min=0, max=2000
  - Spindle Post-Delay ms: "vending.motor.spindle_post_delay_ms", int, min=0, max=2000
  - Modbus Port: "hardware.modbus.port" (text input or just display current value with edit button using TextInputDialog pattern)
  - Modbus Slave Address: "hardware.modbus.slave_address", int, min=1, max=247

For each numeric field: create a tappable row (BoxLayout horizontal) with label on left and value display on right. When tapped, open NumpadDialog, on submit call `coordinator.update_state(path, value)` and update the displayed value. Initialize displayed value from coordinator.get_state or config default.

- name='wizard_sensor'
- step_title="Sensoren"
- step_description="Tursensor-Konfiguration"
- Essential settings (SettingsCard "Tursensor"):
  - Door Sensor GPIO Pin: "hardware.gpio.door_sensor_pin", int, min=0, max=27
  - Active Level: "hardware.gpio.door_sensor_active" - two-option selector (buttons or toggle). Show "LOW" and "HIGH" as tappable buttons, highlight the active selection with CORAL_ACCENT. LOW = door open when pin is LOW (switch closes to ground). HIGH = door open when pin is HIGH.

**server_step.py** - ServerStepScreen (Step 5, step_index=4):

- step_title="Server"
- step_description="Kaufserver und Telemetrie"
- Essential settings (SettingsCard "Kaufserver"):
  - Purchase Server URL: "purchase_server.base_url" (text input - tappable row opens TextInputDialog or inline text input)
  - Machine ID: "system.machine_id" (text input)
- Advanced settings (ExpandableSection "Erweiterte Einstellungen"):
  - Poll Interval (s): "purchase_server.poll_interval_s", float, min=0.1, max=5.0, allow_decimal=True
  - Timeout (s): "purchase_server.timeout_s", float, min=1.0, max=30.0, allow_decimal=True
  - Retry Attempts: "purchase_server.retry_attempts", int, min=0, max=10
  - Telemetry Port: "telemetry.port", int, min=1024, max=65535
  - Telemetry PIN: "telemetry.debug_pin" (text input, initially masked)
  - Check Purchase Endpoint: "purchase_server.endpoints.check_purchase" (text input)
  - Complete Purchase Endpoint: "purchase_server.endpoints.complete_purchase" (text input)

For text input fields: Create a tappable row that shows the current value (or masked for PIN). When tapped, show a dialog or popup with a text input field. On confirm, call coordinator.update_state and update the display. Use the pattern from widgets.py TextInputDialog or implement a simple MDDialog with MDTextField content.

**completion_step.py** - CompletionStepScreen:
- name='wizard_complete'
- This is NOT a BaseWizardStep subclass - it's a simpler Screen
- Inherits directly from `kivy.uix.screenmanager.Screen`
- Layout: centered vertically with:
  - Large checkmark or success icon (use MDLabel with a checkmark unicode character like "✓" in H1 or H2 size, colored CORAL_ACCENT)
  - "Einrichtung abgeschlossen!" (Setup complete!) in H4 font, centered
  - Subtitle: "Die Maschine ist bereit." (The machine is ready.) in Body1, gray text
  - "Fertig" (Finish) button - MDRaisedButton, md_bg_color=CORAL_ACCENT, full width, height="56dp"
  - On "Fertig" press: call coordinator._complete_wizard() to write config, then navigate to customer screen via coordinator.on_complete callback
- Background should match app theme (dark)
- No header, no dots, no skip/back - this is the final screen

IMPORTANT: The completion step is where the atomic write happens. All prior steps only wrote to coordinator.wizard_state dict. The "Fertig" button triggers the actual save.
