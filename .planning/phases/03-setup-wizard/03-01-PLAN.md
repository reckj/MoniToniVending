  <action>
Create the `monitoni/ui/wizard/` package with the following files:

**coordinator.py** - WizardCoordinator class:
- `__init__(self, config_manager, hardware, on_complete)` - takes ConfigManager, HardwareManager, and completion callback
- `self.wizard_state = {}` - in-memory dict for wizard config accumulation
- `self.current_step = 0` - 0-based index
- `self.steps = []` - populated later by register_steps()
- `update_state(path, value)` - set value at dot-notation path in wizard_state dict (build nested dicts from path parts)
- `get_state(path, default=None)` - get value at dot-notation path, return default if not found
- `next_step()` - increment current_step, return step screen name (or call _complete_wizard if past last step)
- `prev_step()` - decrement current_step, return step screen name (or None if at first step)
- `skip_step()` - same as next_step() (skip doesn't save anything for current step, just advances)
- `register_steps(steps_list)` - set self.steps from list of screen name strings
- `get_total_steps()` - return len(self.steps) excluding completion screen
- `_complete_wizard()` - merge wizard_state with existing local.yaml (if exists) via ConfigManager._deep_merge, write atomically using `atomic_yaml_write` from utils.py, reload config via config_manager.load(), invoke on_complete callback. Return 'wizard_complete' screen name.
- For re-run from debug menu: if local.yaml exists, pre-fill wizard_state from current local config values on init. Add `prefill_from_config()` method that loads local.yaml into wizard_state so fields show current values.

**utils.py** - Utility functions and widgets:
- `atomic_yaml_write(yaml_content: str, target_path: str)` - Write YAML atomically using tempfile.mkstemp in same directory as target, os.fdopen to write, os.fsync to flush, os.replace to atomically swap. Wrap in try/finally to clean up temp file on failure. Use pathlib.Path for parent.mkdir(parents=True, exist_ok=True).
- `ExpandableSection(BoxLayout)` - Progressive disclosure widget:
  - `__init__(self, title, **kwargs)` - orientation='vertical', size_hint_y=None
  - `self.is_expanded = False`
  - Toggle button: MDFlatButton with text "▸ {title}" (collapsed) / "▾ {title}" (expanded), theme_text_color='Custom', text_color=CORAL_ACCENT, halign='left', size_hint=(1, None), height="44dp"
  - Content container: BoxLayout orientation='vertical', spacing="8dp", size_hint_y=None, opacity=0 and height=0 when collapsed
  - `toggle()` method: flip is_expanded, update button text arrow, show/hide content by setting opacity and height (bind minimum_height when expanded, 0 when collapsed)
  - `add_content(widget)` - add widget to content_box
  - Bind self.minimum_height to self.setter('height') for proper ScrollView sizing

**__init__.py** - Package init:
- Export WizardCoordinator from coordinator
- Export FirstRunDialog (will be created in Task 2)
- Use lazy imports or try/except to avoid circular import issues

**steps/__init__.py** - Empty package init (steps added in Plan 03-02)
