**In on_start() method (or via Clock.schedule_once in build):**
Add first-run detection:
```python
def _check_first_run(self, dt=None):
    config_manager = get_config_manager()
    if not config_manager.local_config_path.exists():
        self._show_first_run_dialog()
    # If local.yaml exists, just stay on customer screen (normal startup)

def _show_first_run_dialog(self):
    from monitoni.ui.wizard.first_run_dialog import FirstRunDialog
    dialog = FirstRunDialog(
        on_run_wizard=self._start_wizard,
        on_skip=lambda: None  # Stay on customer screen with defaults
    )
    dialog.open()

def _start_wizard(self):
    """Launch wizard from first-run dialog."""
    self.screen_manager.current = 'wizard_hardware'
```

Use `Clock.schedule_once(self._check_first_run, 0.5)` in on_start() to defer dialog display until UI is fully rendered.

**Add new method for launching wizard from debug menu (re-run):**
```python
def start_wizard_from_debug(self):
    """Launch wizard from debug menu (re-run with pre-filled values)."""
    # Pre-fill wizard state from current config
    self.wizard_coordinator.prefill_from_config()
    self.wizard_coordinator.current_step = 0
    self.screen_manager.current = 'wizard_hardware'
```

Import `get_config_manager` at top of file if not already imported, and `from kivy.clock import Clock`.
