<task type="auto">
  <name>Task 2: Create HoldButton and LiveStatusCard widgets</name>
  <files>monitoni/ui/debug_screens/widgets.py</files>
  <action>
Add to the existing widgets.py file:

**HoldButton:**
- MDRaisedButton subclass for hold-to-activate hardware control
- Constructor: `HoldButton(text, on_hold, on_release_hold, **kwargs)`
- on_touch_down: check collide_point first, if hit -> set self._holding=True, call on_hold callback, return True to consume event
- on_touch_up: if self._holding -> set _holding=False, call on_release_hold callback, return True
- on_touch_up must ALSO handle case where touch moves off button (touch.grab) - use grab_current pattern:
  - In on_touch_down, call touch.grab(self) to receive on_touch_up even if finger moves
  - In on_touch_up, check if touch.grab_current is self before processing
  - Call touch.ungrab(self) on release
- Visual feedback: change md_bg_color to coral (#F24033) while held, restore original on release
- Safety: if widget is removed from tree while holding, call on_release_hold in on_pre_leave or cleanup
- Default styling: near-black background, coral while active, size_hint_y=None, height="60dp"

**LiveStatusCard:**
- MDCard subclass showing real-time hardware status
- Constructor: `LiveStatusCard(title, get_status_callback, update_interval=0.5, **kwargs)`
- title: header label for the status section
- get_status_callback: async or sync callable returning list of (label, value, color) tuples
  - e.g., [("Door", "CLOSED", (0, 1, 0, 1)), ("Relay 3", "ON", (1, 0.5, 0, 1))]
- Uses Clock.schedule_interval with lambda for strong reference (avoid weak ref garbage collection)
- Stores self._update_event for cleanup
- on_pre_leave (or a cleanup method): cancel the scheduled event via self._update_event.cancel()
- Status items displayed as horizontal rows: label on left, value on right with colored text
- Near-black background, consistent card styling
- Handle async callbacks: if callback is async, use asyncio.create_task() and update UI via Clock.schedule_once
- Handle errors gracefully: if callback raises, show "ERROR" in red, don't crash the polling loop

**NumpadField:**
- Convenience widget: MDTextField-like display that opens NumpadDialog on tap
- Constructor: `NumpadField(label, config_path, config_manager, allow_decimal, min_value, max_value, on_value_changed=None)`
- Displays current value from config
- On tap/focus: opens NumpadDialog with current value
- On submit: calls update_config_value, updates display, calls on_value_changed if provided
- For risky config paths: shows confirmation dialog before applying
- Disabled text input (no keyboard) - uses on_touch_down to open numpad instead
- Row layout: label on left, value display on right (tappable)
