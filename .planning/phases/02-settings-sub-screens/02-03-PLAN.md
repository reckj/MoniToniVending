This is the KEY feature of the sensor screen - large, prominent display.
- Very large status display taking significant screen space:
  - Large MDLabel showing "TÜR: OFFEN" or "TÜR: GESCHLOSSEN" in H3/H4 font style
  - Color: green (#00FF00) for closed (GESCHLOSSEN), coral (#F24033) for open (OFFEN), gray for disconnected

  - Handle sensor not connected: show "NICHT VERBUNDEN" in gray
- Use strong reference (lambda) for Clock callback to avoid garbage collection
- Store _update_event for cleanup in on_pre_leave

**Card 3: Sensor-Info**
Using SettingsCard with title "Sensor-Info":
- Display current configuration summary:
  - "Pin: GPIO {pin} (BCM)"

  - "Aktiv: {active_state}"
  - "Status: Verbunden / Nicht verbunden"
- This is read-only, updates when config changes

**Reset button:**
- "Werkseinstellungen" - resets hardware.gpio section

**on_pre_leave:**
- Cancel Clock update event via self._update_event.cancel()
- Important for battery/CPU: don't poll sensor when screen not visible

**Key implementation detail:**
- The door status polling must handle async properly. Use pattern:
  ```python
  def _poll_door(self, dt):
      asyncio.create_task(self._async_poll_door())

  async def _async_poll_door(self):
      try:
          state = await self.hardware.sensor.get_door_state()
          Clock.schedule_once(lambda dt: self._update_door_display(state))
      except Exception:
          Clock.schedule_once(lambda dt: self._update_door_display(None))
  ```
