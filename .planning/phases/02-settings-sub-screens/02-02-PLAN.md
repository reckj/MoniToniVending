**Card 2: Motor-Test**
Using SettingsCard with title "Motor-Test":
- Large HoldButton labeled "Motor testen" (prominent, full-width, height="80dp")
  - Hold-to-activate pattern with full motor sequence:
  - on_hold:
    1. Open spindle lock relay (set spindle_lock_relay to True)
    2. After spindle_pre_delay_ms: activate motor relay (set relay_channel to True)
    3. Motor runs while held
  - on_release_hold:
    1. Deactivate motor relay immediately
    2. After spindle_post_delay_ms: close spindle lock relay
  - Use Clock.schedule_once for delays within the sequence
  - Cancel any pending scheduled events on release
  - Store scheduled events in list for cleanup

- Separate HoldButton "Spindel-Schloss testen" - directly controls spindle lock relay only
- Separate HoldButton "Motor direkt testen" - directly controls motor relay only (bypasses spindle sequence)
- These individual test buttons help diagnose whether spindle or motor has issues

**Card 3: Timing-Visualisierung (Timing Visualization)**
- Simple text display showing the complete sequence with current timing values:
  ```
  Sequenz:
  1. Spindel öffnen
  2. Warte {pre_delay}ms
  3. Motor AN (solange gehalten)
  4. Motor AUS
  5. Warte {post_delay}ms
  6. Spindel schließen
  ```

- Use MDLabel with monospace-ish layout for clarity

- "Werkseinstellungen" at bottom
