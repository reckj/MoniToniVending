Create `monitoni/ui/debug_screens/network_screen.py` with class `NetworkSettingsScreen(BaseDebugSubScreen)`.

Constructor: `__init__(self, hardware, config_manager, navigate_back=None, **kwargs)`
- Set title = "Netzwerk" (German)

**Card 1: Server-Verbindung (Server Connection)**
Using SettingsCard with title "Server-Verbindung":
- Tappable text field for "Server-URL" -> config path "purchase_server.base_url"
  - This is a string (URL), so use a simple text edit dialog (not numpad)
  - On tap: open MDDialog with MDTextField for URL input, OK/Cancel buttons
  - On change: save to config
- Tappable text field for "Maschinen-ID" -> config path "system.machine_id"
  - String input, same text dialog pattern
- NumpadField for "Abfrage-Intervall (s)" -> config path "purchase_server.poll_interval_s" (decimal, min=0.1, max=10.0)
- NumpadField for "Timeout (s)" -> config path "purchase_server.timeout_s" (decimal, min=1.0, max=30.0)
- NumpadField for "Wiederholungen" -> config path "purchase_server.retry_attempts" (min=0, max=10)
- Toggle for "Server aktiviert" -> config path "purchase_server.enabled"

**Card 2: Verbindungs-Test (Connection Test)**
Using SettingsCard with title "Verbindungs-Test":
- Large "Verbindung testen" button (full width, height="70dp", coral accent)
- On press:
  1. Show "Teste..." status
  2. Try to reach purchase_server.base_url with a simple HTTP GET (timeout=5s)
  3. Use asyncio.create_task for non-blocking
  4. On success: show green "Verbunden - Server erreichbar" with response time
  5. On failure: show red "Fehler - {error_message}"
  6. Use requests or aiohttp (check what's available in the project)
  - Since we can't guarantee aiohttp is available, use urllib.request in a thread:
    ```python
    import urllib.request
    # Run in thread to avoid blocking UI
    ```
  - Or use the existing purchase_client pattern if available
- Result display: MDLabel showing test result, persists until next test

**Card 3: Netzwerk-Status (Network Status)**
Using SettingsCard with title "Netzwerk-Status":
- Display WiFi status by reading system info:
  - Get WiFi SSID: subprocess or read /proc/net/wireless (handle gracefully if not available)
  - Get IP address: use socket.gethostbyname(socket.gethostname()) or parse ifconfig/ip output
  - Simpler approach: use `socket.getaddrinfo(socket.gethostname(), None)` or `netifaces` if available
  - Fallback: just show "Nicht verfügbar" if system info can't be read
- Display rows:
  - "WiFi: {ssid}" or "WiFi: Nicht verbunden"
  - "IP-Adresse: {ip}" or "IP: Nicht verfügbar"
  - "Server: {base_url}"
  - "Status: Verbunden/Getrennt" (based on last connection test or health check)
- Auto-refresh every 5s via Clock.schedule_interval

**Card 4: Telemetrie (Telemetry)**
Using SettingsCard with title "Telemetrie":
- Display telemetry server config (read-only informational):
  - "Telemetrie-Port: {port}" from config.telemetry.port
  - "Debug-PIN: ****" (masked) with "Anzeigen" toggle button to reveal
  - "Telemetrie aktiviert: JA/NEIN" from config.telemetry.enabled

**Reset button:**
- "Werkseinstellungen" - resets purchase_server and system sections

**Helper: TextInputDialog**
- Since network screen needs string input (URLs, IDs), create a simple helper:
  - MDDialog with MDTextField for text entry
  - Title, initial value, on_submit callback
  - Can be added to widgets.py or kept local to network_screen.py
  - Add it to widgets.py since LED screen also needs it for IP address

**on_pre_leave:**
- Cancel network status polling Clock event

**Implementation notes:**
- Getting IP/WiFi info: use subprocess.run with timeout to avoid hanging
  - `ip -4 addr show wlan0` or `hostname -I` for IP
  - `iwgetid -r` for WiFi SSID
  - Catch all exceptions, show "Nicht verfügbar" on failure
  - These are quick system calls, safe to run synchronously
