**1b. Create `maintenance_screen.py`:**

A new BaseDebugSubScreen following established patterns.

**Screen structure:**
- Title: "Wartung & Status"
- Constructor: `__init__(self, config_manager, hardware, navigate_back=None, **kwargs)` (matching all other sub-screens)

**UI layout (2 SettingsCards):**

1. **Operating Mode Card** ("Betriebsmodus"):
   - BoxLayout row with MDLabel "Wartungsmodus:" (size_hint_x=0.7) and MDSwitch
   - CRITICAL KivyMD 1.2.0 quirk: Set MDSwitch.active via `Clock.schedule_once(lambda dt: setattr(self.maintenance_toggle, 'active', current_state))` — do NOT set active in constructor or pass active=True
   - Bind `on_active` callback to `on_maintenance_toggle` method
   - On toggle: call `update_config_value(self.config_manager, "system.maintenance_mode", value)`
   - If save fails, revert toggle using Clock.schedule_once
   - MDLabel info text below: "Im Wartungsmodus ist der Verkauf gesperrt." (font_style='Caption', theme_text_color='Secondary')

2. **Machine Status Card** (LiveStatusCard):
   - Title: "Maschinenstatus"
   - Use `LiveStatusCard(title="Maschinenstatus", get_status_callback=self.get_machine_status, update_interval=2.0)`
   - `get_machine_status` is an `async def` method returning `List[Tuple[str, str, Tuple]]`:
     - ("Modus:", "WARTUNG" or "Betrieb", ERROR_RED or green)
     - ("Relais:", "OK" or "FEHLER", green or ERROR_RED) — wrap in try/except
     - ("LEDs:", "OK" or "OFFLINE", green or yellow) — wrap in try/except
     - ("Tür:", "OFFEN" or "Geschlossen", yellow or grey) — wrap in try/except
     - ("Audio:", "Aktiv" or "Deaktiviert", green or grey) — read from config
   - Use colors: green = (0, 0.8, 0, 1), yellow = (1, 1, 0, 1), grey = (0.5, 0.5, 0.5, 1)
   - Each hardware check wrapped in try/except to handle disconnected hardware gracefully

**Key implementation details:**
- Import: `from monitoni.ui.debug_screens.widgets import SettingsCard, LiveStatusCard, update_config_value, CORAL_ACCENT, NEAR_BLACK, ERROR_RED`
- Import: `from kivymd.uix.selectioncontrol import MDSwitch`
- Import: `from kivy.clock import Clock`
- The `update_config_value` returns `(success, needs_confirm)` tuple — check success before proceeding

Add a maintenance mode check that prevents product selection when maintenance mode is active.

**Integration point: `_on_level_selected` method (around line 585):**

At the very beginning of `_on_level_selected`, before the idle state check, add a maintenance mode check:

```python
def _on_level_selected(self, button: ProductButton):
    # Check maintenance mode first
    if self.app_config.system.maintenance_mode:
        self.set_status("Maschine wird gewartet", (1, 0.5, 0, 1))  # Orange
        return

    # Existing idle state check...
    if not self.state_machine.is_idle():
        ...
```

**Why this approach (not a full-screen takeover):**
- The customer screen already has a `set_status()` method for showing messages
- Blocking at the selection handler is simpler and safer than replacing the entire UI
- Products remain visible but non-functional — operator might want customers to see what's available
- The status message provides clear feedback
- If maintenance mode is toggled off, next tap works immediately (no screen rebuild needed)

**Access the config:** The `app_config` is already passed to CustomerScreen constructor as a `Config` object. Need to verify it reflects live config changes. If `app_config` is a snapshot, instead use `get_config_manager().config.system.maintenance_mode` to read the live value.

Check how `app_config` is used elsewhere in customer_screen.py. If it reads from the Config pydantic model which is backed by ConfigManager's live config, it will reflect changes. If it's a static snapshot, use `get_config_manager()` instead.

Read the config module to determine the right approach. The safest approach is:
```python
from monitoni.core.config import get_config_manager

# In _on_level_selected:
config_mgr = get_config_manager()
if hasattr(config_mgr.config, 'system') and hasattr(config_mgr.config.system, 'maintenance_mode'):
    if config_mgr.config.system.maintenance_mode:
        self.set_status("Maschine wird gewartet", (1, 0.5, 0, 1))
        return
```

Use `hasattr` checks for graceful fallback if config hasn't been updated yet (e.g., running with old local.yaml that doesn't have maintenance_mode).
