  <action>
Create `qr_management_screen.py` as a new BaseDebugSubScreen following established patterns from relay_screen.py, audio_screen.py, etc.

**Screen structure:**
- Title: "QR Code Verwaltung"
- Constructor signature: `__init__(self, config_manager, hardware, navigate_back=None, **kwargs)` (matching all other sub-screens)

**UI layout (3 SettingsCards):**

1. **Level Selection Card** ("Level auswählen"):
   - NumpadField for selecting product level (1 to `config_manager.config.vending.levels`)
   - On value change, update preview

2. **QR Preview Card** ("Aktuelle QR Code"):
   - Kivy `Image` widget (size_hint=(1, None), height="300dp", allow_stretch=True, keep_ratio=True) showing current QR for selected level
   - MDLabel status text below preview indicating source: "Level N (Generiert)", "Level N (Custom)", or "Level N (Kein QR Code)"

3. **Actions Card** ("Aktionen"):
   - "QR Code generieren" button (MDRaisedButton, md_bg_color=CORAL_ACCENT):
     - Opens TextInputDialog with title "Zahlungslink URL eingeben"
     - Default value: current base URL from customer_screen pattern `https://www.monitoni.zhdk.ch?level={N}`
     - On submit: generate QR code using `qrcode.QRCode(version=1, error_correction=qrcode.constants.ERROR_CORRECT_M, box_size=10, border=4)`, save to `assets/qr_codes/level_{N}.png`, refresh preview
   - "Von USB hochladen" button (MDRaisedButton, md_bg_color=NEAR_BLACK):
     - Uses `MDFileManager(exit_manager=..., select_path=..., preview=True, ext=['.png', '.jpg', '.jpeg'])`
     - Opens at `/media/{USER}/` if exists, fallback to `/media/`
     - On file select: copy with `shutil.copy2()` to `assets/qr_codes/custom_level_{N}.png`, refresh preview
   - "Custom QR löschen" button (MDRaisedButton, md_bg_color=ERROR_RED):
     - Only active if custom QR exists for selected level
     - Shows MDDialog confirmation ("Custom QR Code für Level N wirklich löschen?")
     - On confirm: delete `custom_level_{N}.png`, refresh preview

**Key implementation details:**
- QR directory: `Path("assets/qr_codes")` with `mkdir(parents=True, exist_ok=True)` in constructor
- Preview update logic: check `custom_level_{N}.png` first (precedence), then `level_{N}.png`, then show empty state. This matches the existing pattern in customer_screen.py `_load_qr_code()`.
- After generating or uploading, force Kivy image reload by setting `.source = ""` then back to the path, or calling `.reload()` on the Image widget
- Use `show_confirm_dialog` pattern from widgets.py for delete confirmation
- Error handling: wrap QR generation and file operations in try/except, show MDDialog with error message on failure
- Import color constants: CORAL_ACCENT, NEAR_BLACK, ERROR_RED from widgets.py
- Import TextInputDialog from widgets.py for URL input
- Import NumpadField from widgets.py for level selection
